# 交易者首次使用 Kronos — 体验日志

> 模拟时间：2026-05-05
> 角色：张伟，3 年 crypto 现货交易经验，会 Python + pandas，未用过量化框架
> 环境：全新 git clone + venv，无数据，无 DeepSeek

---

## 安装阶段

```
$ git clone https://github.com/liuyejinghong/Kronos.git
$ cd Kronos
$ pip install uv
$ uv sync --dev
```

装完 200+ 个包，花了 3 分钟。没有报错，OK。

启动后弹出 ASCII 边框和中文菜单。第一印象：这像个开发工具，不太像产品。

---

## 问题 1：没有数据，也不知道怎么获取

```
📊 环境状态
  📭 未找到本地数据
  ⚠️  DeepSeek: 未配置（Agent 研究功能受限）
```

**困扰**：我是来跑 BTC 策略的。没有数据？那我该做什么？提示说运行 `kronos quickstart` 或者 `kronos data sync`。

**困惑**：
- `quickstart` 是什么？它会生成什么样的数据？是真实数据还是模拟数据？
- `data sync` 告诉我是"同步真实行情"，但它从哪同步？需要 API key 吗？要多久？
- 为什么不能直接在 Agent 里选"帮我准备数据"？

---

## 问题 2：选"查看行情"，什么都看不到

```
📈 可用行情数据
未找到本地数据
```

**困扰**：我点了"查看行情"是想看价格走势。结果告诉我没数据，然后弹回菜单。

**期待**：即使没有数据，也应该告诉我"你还没有数据，我可以帮你——[1]生成模拟数据（7 天 BTC 历史），[2]连接 Binance 拉取真实数据"。

---

## 问题 3：候选策略完全看不懂

```
📋 候选策略池
  共 12 个候选策略
   1. 指标 spread regime  [trend_momentum]  → retired
   2. 信号持续性密度  [trend_momentum]  → retired
   3. 趋势回撤容忍度  [trend_momentum]  → retired
```

**困扰**：这些是什么？

- "指标 spread regime" — 完全听不懂。我是交易者，我关心的是"这个策略在 BTC 上赚钱吗？"
- `retired` 是什么意思？这些策略已经废了？那为什么还在列表里？
- `trend_momentum` 是什么？是一种因子类型吗？因子又是什么？
- 这些跟我的 BTC 交易有什么关系？系统怎么知道我关注什么品种？

**根本问题**：候选池暴露的是内部研发概念（因子名、生命周期状态、家族名），不是交易者能理解的策略描述。

---

## 问题 4：Agent 不 Agent

标题写着 "Kronos Agent — 交互模式"，但实际体验是：

```
你想做什么？
  [1] 📈 查看可用行情数据
  [2] 🔬 开始一轮研究
  [3] 📋 查看候选策略池
  [4] ⚙️  配置模型
  [5] 📜 查看历史运行
  [0] 🚪 退出
```

**困惑**：
- 这不是 Agent，这是**菜单**。菜单不是对话。
- 真正的 Agent 应该问"你好，想研究什么？"然后我回答"帮我看看 BTC 的趋势策略"，而不是我自己去研究菜单里的选项是什么意思。
- 我输入数字，Agent 回吐文本，然后弹回同一个菜单。没有对话感，没有上下文。

---

## 问题 5：没有任何引导我"开始第一次研究"

Agent 知道我没数据、没配置模型、没跑过任何研究。但它什么都没主动做。

**期待**：Agent 应该主动说"你第一次来，我帮你准备。等我 30 秒——生成数据→检查候选→从最热门的 3 个开始。要不要试试？"然后我只需要按 Y。

而不是抛给我 5 个选项让我自己去选。

---

## 问题 6：候选状态用英文 enum

候选状态显示的是 `retired`、`observe`、`redesign`。这是代码里 field name = `value` 的 bug——应该用中文标签"淘汰"、"观察"、"候选改造"。

**复现**：`c.lifecycle_state` 返回 `CandidateLifecycleState.RETIRED`，`.value` 是 `"retired"` 而不是中文标签。

---

## 总结：一个交易者需要什么

| 交易者的心智模型 | Kronos 当前的呈现 |
|---|---|
| "我要一个策略跑 BTC" | 菜单里有 5 个选项，不知道先点哪个 |
| "这个策略过去表现怎么样" | 候选列表里是技术术语 |
| "Agent 帮我分析一下" | 实际上是我自己在操作 |
| "我想要 TradingView 那样的图表" | 只有 CLI 文字 |
| "我该怎么做" | 没有主动引导 |

**核心差距**：Kronos 是为开发者设计的，不是为交易者设计的。菜单里的概念（候选、因子、研究循环）需要大量的领域知识才能理解。Agent 标题写着"交互模式"但没有对话能力——只是把 CLI 子命令换成了数字菜单。
